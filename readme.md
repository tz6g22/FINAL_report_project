# 实验流程说明（从数据到结论）

这份文档只讲一件事：这个项目的实验是怎么一步一步跑起来的。  
重点不是代码细节，而是整个实验链路。

## 实验目标

这个项目要比较两种模型在同一套数据上的表现：

- `standard`
- `attnres`

核心思想是：  
同样的数据、同样的训练条件，分别训练两边，然后看谁结果更好、更稳定。

## 整体流程

1. 先确定实验参数和输出目录。  
2. 读入 TinyStories 数据。  
3. 把文本处理成模型能训练的小段数据。  
4. 开始训练（按步推进，不断更新）。  
5. 训练中定期做验证和保存。  
6. 训练结束写出总结。  
7. 对比两种模型的结果，得到实验结论。

## 分步解释

### 第一步：实验一开始准备了什么

入口主要有两个：

- `launch_dual_ddp.py`：同时启动两边训练（常用）
- `mini_gpt_attnres/run_tinystories.py`：按顺序跑

这一步会准备：

- 要训练哪种模型
- 训练多少步
- 每次喂多少数据
- 输出目录放哪里

如果用双任务入口，会把两边分开：

- `standard` 放一组显卡
- `attnres` 放另一组显卡

### 第二步：数据进入后先做什么

数据入口在 `mini_gpt_attnres/data.py` 和 `mini_gpt_attnres/data_tinystories.py`。  
会做三件事：

1. 读取 TinyStories 的训练和验证文本  
2. 把文本转成数字序列  
3. 按固定长度切成一段一段训练样本

结果是两份可直接喂给模型的数据：

- 训练数据
- 验证数据

### 第三步：中间经历了哪些主要阶段

训练主流程在 `mini_gpt_attnres/train.py`，大致是循环执行：

1. 取一批训练数据  
2. 算当前误差  
3. 更新模型参数  
4. 记录这一步日志

并且每隔一段步数，会插入两件事：

- 验证：看当前模型在验证数据上的表现
- 保存：把当前模型状态写盘

### 第四步：每个阶段产出了什么

每个模型都会在自己的目录里产出：

- `metrics_模型名.jsonl`：每一步的训练记录
- `ckpt_模型名_step_xxxxxx.pt`：阶段保存
- `ckpt_模型名_last.pt`：最后保存
- `summary_模型名.json`：最终汇总

### 第五步：后面的步骤如何依赖前面的结果

依赖关系是很直观的：

- 没有处理好的数据，就进不了训练
- 没有训练中的参数更新，就不会有后续验证结果
- 没有阶段保存，就无法回看中间状态
- 没有最终汇总，就没法做最后对比

也就是说，后一步都建立在前一步产物上。

## 对照实验怎么做

对照实验和主实验流程几乎一样，区别只在模型不同：

- 主线一：`standard`
- 主线二：`attnres`

其余条件尽量保持一致：

- 同一份数据
- 同样的训练步数
- 同样的批量大小
- 同样的上下文长度

双任务入口会让两边同时跑，方便直接对照。

## 最终结果怎么来的

最后的结论不是看一条日志，而是综合三类结果：

1. 每步训练记录（看趋势是否稳定）  
2. 验证结果（看效果好坏）  
3. 最终汇总文件（看终局指标）

把 `standard` 和 `attnres` 的这些结果放在一起比较，就得到本次实验结论。

## 用大白话再总结一遍

这套实验就是：

先把故事数据整理好，喂给两种模型去学；  
学习过程中一直记账、定期考试、定期存档；  
最后把两份成绩单摆在一起比。

最核心的一条主线只有一句话：  
**同条件训练，两边对照，用真实训练记录得出结论。**
